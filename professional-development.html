Professional Development

Career planning, employability development, certifications, GitHub progress and long-term preparation for cloud, DevOps and SRE roles.

Career Goal

My long-term career goal is to move into cloud, DevOps or site reliability engineering. I am interested in roles that combine programming, automation, infrastructure, monitoring, troubleshooting and reliable systems.

At this stage, I am building foundations through my Computer Science degree, personal projects, GitHub portfolio work, freeCodeCamp practice and early exposure to Linux, cloud and automation concepts.

Year 1: Foundations and Early Projects

The first year focuses on building strong foundations in programming, web development, GitHub and basic Linux skills.

  • Develop a personal e-portfolio using HTML, CSS and GitHub Pages.
  • Build JavaScript mini projects such as a calculator and quiz app.
  • Create a Bash or Python file organiser to practise scripting.
  • Push code to GitHub regularly and document project progress.
  • Begin learning Linux basics and command-line workflows.

Year 2: Cloud, Full Stack and Internship Preparation

The second year focuses on expanding from basic programming into cloud, APIs, containers, infrastructure and deployment.

  • Build a Node.js API and connect it to a front-end project.
  • Dockerise a project using Dockerfiles and containers.
  • Create a Terraform deployment using cloud resources.
  • Build CI/CD pipelines using GitHub Actions.
  • Develop a monitoring dashboard using Prometheus and Grafana.
  • Apply for internships or placements related to IT, cloud or software support.

Years 3 and 4: SRE, Kubernetes and Graduate Preparation

Later years focus on more advanced reliability engineering skills, portfolio evidence and graduate job preparation.

  • Build a full-stack app deployed with containers and Kubernetes.
  • Create an SRE lab repository to document infrastructure projects.
  • Combine CI/CD, Kubernetes and monitoring into one project.
  • Complete a final-year project using cloud, CI/CD and observability tools.
  • Prepare for graduate SRE, DevOps, cloud or IT infrastructure roles.

Skills Matrix and Action Plan

Current Skills

  • Programming: Python basics, JavaScript basics, functions, loops, conditionals and data structures.
  • Web development: HTML, CSS, responsive design, GitHub Pages and project presentation.
  • Problem-solving: debugging, testing, input validation and breaking tasks into smaller steps.
  • Research: academic reading, literature searching, referencing and reflective writing.
  • Communication: customer-facing experience, peer discussion, written explanations and feedback responses.
  • Employability: CV development, LinkedIn profile building, GitHub portfolio work and career planning.

Employability Evidence

My customer-facing work experience has helped me develop communication, patience, teamwork and problem-solving skills. These are useful for IT support, technical troubleshooting and future engineering roles, where clear communication and reliability matter alongside technical ability.

My university work and personal projects show my ability to learn independently, apply feedback and gradually build more complex technical skills. This portfolio is part of that development, as it records my progress and creates evidence for future placements, internships and graduate applications.

Certification Plan

  • Year 1: freeCodeCamp Responsive Web Design and JavaScript practice, plus early AWS Cloud Practitioner study after basic cloud exposure.
  • Year 2: Linux Foundation Certified IT Associate or equivalent Linux basics, followed by Docker hands-on labs after container projects.
  • Year 3: AWS Solutions Architect Associate and Terraform Associate after completing cloud and infrastructure-as-code projects.
  • Year 4: SRE fundamentals, GitHub Actions or DevOps micro-certifications, and optional Kubernetes certification if my practical skills are ready.

Development Habits

  • Push at least one meaningful commit to GitHub each week.
  • Document projects clearly with README files, screenshots and live links.
  • Spend regular time on freeCodeCamp, Python practice and cloud fundamentals.
  • Review CV, LinkedIn and portfolio content monthly.
  • Apply for placements, internships and entry-level IT opportunities when relevant.
  • Use feedback from tutors and peers to improve academic and technical work.

Short-Term Action Plan

  • Finish and publish this e-portfolio using GitHub Pages.
  • Upload the cozy calculator project to GitHub with a clear README file.
  • Complete the JavaScript quiz app and link it to the portfolio.
  • Continue strengthening Python through university assignments and small projects.
  • Begin structured Linux and cloud learning before applying for technical placements.

Reflection

This professional development plan gives me a clearer route from student projects to technical employability. Instead of only focusing on individual assignments, I am building a visible record of skills, projects and career preparation.

The plan also helps me avoid trying to learn everything at once, which is a classic student self-sabotage ritual. By linking certifications and projects to each year of study, I can build skills gradually and create stronger evidence for future cloud, DevOps and SRE opportunities.
