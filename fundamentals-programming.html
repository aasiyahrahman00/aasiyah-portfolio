Fundamentals of Programming

Module record for procedural programming, Python, control flow, functions, data structures, debugging and testing.

Module Overview

This module introduced fundamental programming concepts using Python, with a focus on procedural programming. It covered variables, data types, control structures, functions, lists, dictionaries, input and output, memory concepts, abstraction, debugging and testing.

The module helped me move from learning isolated programming concepts to applying them in practical programs, including calculators, quizzes, a financial transaction tracker and a text-based adventure game.

Learning Outcomes

  • Understand variables, data types, control structures, functions and arrays.
  • Design algorithms to solve simple problems.
  • Implement algorithms using Python.
  • Analyse code through debugging and testing.
  • Apply procedural programming concepts in practical applications.

The mid-module assignment required me to create a Python program that allowed users to track income and expenses. The program used variables, control flow, functions and a list of dictionaries to store transaction data.

End of Module Assignment: Text-Based Adventure Game

The end-of-module assignment required me to design and implement a Python-based text adventure game. My game, The Bunny and the Golden Key, involves navigating through connected rooms to find a mythical Golden Key and escape to banana paradise before the burrow collapses.

  • Due date: 25 May 2026
  • Weighting: 70%
  • Status: Submitted / awaiting grade
  • Main data structure: nested dictionary for rooms, exits, items, traps and hints
  • Other structures: lists for inventory, collected rooms, triggered traps and warnings
  • View Adventure Game Repository

The game includes six connected rooms, unique descriptions, collectible items, hidden traps, hints, a limited move counter, inventory tracking and a replay system. Functions were used to organise the game mechanics, including movement, item collection, trap checking, inventory display, hints, collapse warnings and win/loss conditions.

Debugging and testing were carried out throughout development. This included checking invalid directions, incorrect item collection, empty inventory states, the replay loop, trap conditions and the move countdown losing condition.

Reflection

This module helped me understand how programming concepts connect together in real programs. At the beginning, I focused mainly on getting code to run. As the module progressed, I became more aware of structure, readability, input validation and how early design decisions affect later functionality.

The financial transaction program helped me understand the importance of choosing suitable data structures. The text-based adventure game helped me apply functions, dictionaries, lists, loops and conditionals in a more creative and complex program.
